## Task Tracker

Добро пожаловать в проект Task Tracker! Это Java-приложение для управления задачами, которое позволяет создавать, обновлять и отслеживать задачи, эпики и подзадачи с поддержкой истории просмотров, временных приоритетов и HTTP API для взаимодействия. Проект развивался через несколько спринтов, добавляя функции вроде сохранения в файл, проверки пересечений задач и всестороннего тестирования.

## Описание проекта

Task Tracker — это система управления задачами, вдохновленная инструментами вроде Jira или Trello. Пользователи могут организовывать свои задачи, разбивать большие задачи на подзадачи, отслеживать прогресс и просматривать историю. Приложение поддерживает хранение данных в памяти или файлах, проверку временных конфликтов и доступ через HTTP API для интеграции с другими системами.

Проект построен с использованием принципов ООП: интерфейсы для абстракции, наследование для типов задач и менеджеров. Он покрыт модульными тестами для надежности.

## Основная функциональность

### Типы задач:
- **Задача (Task)**: Обычная задача с названием, описанием, статусом (NEW, IN_PROGRESS, DONE), длительностью и временем начала.
- **Эпик (Epic)**: Большая задача, состоящая из подзадач. Статус и время рассчитываются на основе подзадач.
- **Подзадача (Subtask)**: Подзадача, связанная с эпиком.

### Менеджеры задач:
- **InMemoryTaskManager**: Хранит данные в оперативной памяти.
- **FileBackedTaskManager**: Автоматически сохраняет состояние в CSV-файл и восстанавливает из него.
- **HistoryManager**: Управляет историей просмотров задач (последние просмотренные задачи без дубликатов, неограниченного размера).

### Приоритеты и время:
- Задачи сортируются по времени начала (startTime).
- Проверка на пересечение задач по времени выполнения.
- Расчет времени окончания (endTime) на основе startTime и длительности.

### История просмотров:
- Отслеживает последние просмотренные задачи.
- Удаляет дубликаты и поддерживает операции O(1).

### HTTP API:
- Сервер на порту 8080 для взаимодействия через HTTP-запросы.
- Поддерживает методы GET, POST, DELETE для задач, подзадач, эпиков, истории и приоритетов.

### Тестирование:
- Полное покрытие модульными тестами с JUnit.
- Тесты для менеджеров, истории, API и граничных случаев.

## Архитектура проекта

### Интерфейсы:
- **TaskManager**: Основные методы для управления задачами.
- **HistoryManager**: Методы для работы с историей просмотров.

### Классы:
- **Task, Epic, Subtask**: Модели задач с полями id, name, description, status, duration, startTime.
- **InMemoryTaskManager**: Реализация менеджера в памяти.
- **FileBackedTaskManager**: Реализация с сохранением в файл (наследуется от InMemoryTaskManager).
- **InMemoryHistoryManager**: Реализация истории с использованием связанного списка и HashMap.
- **HttpTaskServer**: HTTP-сервер для API.
- **Managers**: Утилитарный класс для создания менеджеров.

### Зависимости:
- Java 11+.
- JUnit 5 для тестирования.
- Gson для обработки JSON в API.

Проект использует паттерны вроде фабрики (Managers), наследования и полиморфизма для гибкости.
